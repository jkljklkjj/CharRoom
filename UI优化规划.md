# CharRoom UI 优化规划

## 1. 目标

本规划用于指导 `CharRoom` 客户端的 UI/交互优化，目标是：

1. 提升聊天主流程的稳定性和可用性。
2. 统一视觉语言与交互文案，降低认知负担。
3. 为后续功能扩展（如会话管理、Agent 体验增强）预留结构。

## 2. 当前问题概览

### 2.1 交互与体验

1. 发送态未与网络回调严格绑定，存在重复发送风险。
2. 群聊输入未限制空消息。
3. 在线状态通过修改用户名字符串显示，存在展示污染风险。
4. 移动布局下进入会话后返回路径不明显。

### 2.2 视觉与一致性

1. 主题仍为默认 MaterialTheme，品牌感弱。
2. 中英文文案混用，整体语言风格不统一。
3. 消息气泡样式较基础，信息层级（时间、状态）不够清晰。

### 2.3 工程与可维护性

1. `UserList` 中仍有调试按钮与日志输出。
2. 会话选择存在重复赋值代码。
3. UI 状态主要依赖全局可变状态，后续扩展成本较高。

## 3. 优化范围与优先级

## P0（本周，先做）

1. 修复输入与发送流程稳定性。
2. 清理调试入口与重复逻辑。
3. 统一关键页面文案风格（先中文统一）。

## P1（1-2 周）

1. 聊天页与群聊页视觉升级（气泡、状态、时间、顶部栏）。
2. 会话列表可视化增强（在线状态标识、最近消息预留位）。
3. 登录页交互优化（错误提示、加载态、记住我安全策略）。

## P2（2-4 周）

1. 引入更清晰的状态分层（ViewModel/StateFlow）。
2. Agent 助手体验增强（快捷提问、上下文开关、回复卡片化）。
3. 主题系统完善（亮暗主题、品牌配色、字体层级）。

## 4. 分模块任务清单

### 4.1 聊天主界面

文件：`src/commonMain/kotlin/component/ChatApp.kt`

1. 去掉重复赋值 `selectedUser = user`。
2. 把在线状态从“用户名字符串拼接”改为结构化字段。
3. 在移动布局增加返回入口和顶部导航状态。

### 4.2 私聊页面

文件：`src/commonMain/kotlin/component/ChatScreen.kt`

1. 发送按钮与键盘发送统一走同一发送函数。
2. `isSending` 与真实回调绑定，防止连击重复提交。
3. 增加消息气泡最大宽度、时间显示、失败态样式一致化。

### 4.3 群聊页面

文件：`src/commonMain/kotlin/component/GroupChatScreen.kt`

1. 增加空消息校验与长度限制。
2. 与私聊页对齐发送态、重发态交互。
3. 群消息中增加发送者展示和时间布局优化。

### 4.4 用户列表

文件：`src/commonMain/kotlin/component/UserListScreen.kt`

1. 移除 `Add Test Message` 按钮（或改为 debug 开关）。
2. 去掉生产环境 `println`。
3. 预留会话项扩展位：未读角标、最后消息预览。

### 4.5 登录/注册

文件：`src/commonMain/kotlin/component/LoginRegisterApp.kt`

1. 文案统一为中文（或统一英文，二选一）。
2. 增加提交中禁用与错误提示映射。
3. 替换明文本地密码保存策略（至少只保存账号，不保存明文密码）。

### 4.6 新增联系人弹窗

文件：`src/commonMain/kotlin/component/AddUserOrGroupDialog.kt`

1. 按钮文案与状态提示统一。
2. 添加输入格式校验（仅数字账号/群号）。
3. 成功后交互反馈增强（高亮新会话项或自动选中）。

## 5. 视觉规范建议

1. 建立统一颜色变量：主色、成功、失败、在线、离线。
2. 建立统一间距标尺：4/8/12/16/24。
3. 统一圆角和阴影层级，避免页面元素风格割裂。
4. 统一文本层级：标题、正文、辅助信息、状态提示。

## 6. 验收标准（第一阶段）

1. 私聊/群聊不能发送空白消息。
2. 连续点击发送不会造成重复提交。
3. 用户列表不包含调试按钮与调试打印。
4. 在线状态展示不再依赖用户名字符串拼接。
5. 登录/注册和添加弹窗文案风格统一。

## 7. 里程碑建议

1. M1（2-3 天）：P0 完成并回归测试。
2. M2（1 周）：P1 UI 改版完成。
3. M3（2-4 周）：P2 架构与扩展能力落地。

## 8. 风险与注意事项

1. 发送流程改造需避免影响现有 Netty 回包处理。
2. 状态结构调整要兼顾 Android/Desktop 多端一致性。
3. Agent 会话体验升级需与后端响应格式保持一致。
